# 軟體規格書：Threads/IG 內容監控通知系統 (Project: Monitor-Sync)

## 1. 專案概述
本系統旨在監控特定社群媒體帳號之最新貼文，並在檢測到新貼文時，透過通訊軟體即時通知使用者。系統核心目標為：穩定性、低資源消耗、零帳號封禁風險。

## 2. 功能需求 (Functional Requirements)
- **資料擷取 (Crawler)**：定期抓取目標帳號 (Threads 與 Instagram) 頁面，解析貼文內容與連結。
- **狀態管理 (State Management)**：記錄最後一次監控到的貼文 ID，防止重複通知。
- **通知機制 (Alerting)**：透過 Discord Webhook 將最新貼文的內容與網址推送到指定的 Discord 頻道，方便使用者點擊連結前往進行點讚或留言。
- **隨機排程 (Jittered Scheduling)**：模擬人類檢查頻率，防止被視為機器人。

## 3. 技術規格 (Technical Requirements)
- **語言**：Node.js
- **核心庫**：
  - `axios`：發送 HTTP 請求。
  - `cheerio`：解析 HTML DOM。
  - `node-cron`：排程執行 (預設每 30 分鐘執行一次)。
  - `dotenv`：管理環境變數 (如 Discord Webhook URL)。
- **環境要求**：支援 Node.js v18+，可部署於 Docker 或本地服務器。

## 4. 資料庫架構 (Data Schema)
我們僅需儲存各平台與帳號的最新狀態，建議使用簡單的 JSON 檔案 (`state.json`)：

```json
{
  "threads": {
    "target_account_1": {
      "last_post_id": "1234567890",
      "last_checked_at": "2026-06-23T20:41:00Z"
    }
  },
  "instagram": {
    "target_account_2": {
      "last_post_id": "0987654321",
      "last_checked_at": "2026-06-23T20:41:00Z"
    }
  }
}
```

## 5. 系統工作流程 (Workflow)
1. **啟動監控**：`node-cron` 觸發任務 (設定約 30 分鐘為基準，加入 `Math.random()` 偏移量)。
2. **發送請求**：使用 `axios` 帶入正確的 User-Agent 獲取目標 HTML。
3. **解析資料**：使用 `cheerio` 提取最新的貼文區塊及連結。
4. **比對與判斷**：將抓到的貼文 ID 與狀態檔 (`state.json`) 中的 `last_post_id` 比對。
   - 若 ID 相同或已存在：不處理。
   - 若 ID 不同 (有新文章)：執行通知流程，並更新 `last_post_id`。
5. **通知使用者**：呼叫 Discord Webhook API，將擷取到的「文章內容」與「貼文連結」發送至 Discord 頻道。

## 6. 開發時程建議
- **階段一**：環境搭建與請求測試 (建立 `axios` 請求，成功印出 Threads/IG 目標頁面 HTML)。
- **階段二**：資料解析 (使用 `cheerio` 抓取不同平台的貼文內容、ID 與連結)。
- **階段三**：狀態與比對邏輯 (實作讀寫 JSON 狀態檔的機制)。
- **階段四**：排程與通知整合 (加入 `node-cron` 定時機制，與 Discord Webhook 串接)。
